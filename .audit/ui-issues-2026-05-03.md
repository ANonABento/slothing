### 1. [CRITICAL] Hardcoded status badge colors — breaks theme consistency & dark mode
**Files:**
- `src/app/(app)/dashboard/page.tsx:502-518` — `StatusBadge` local component uses hardcoded `bg-violet-100 text-violet-600 dark:bg-violet-900/30` etc.
- `src/app/(app)/opportunities/[id]/page.tsx:57-67` — `STATUS_BADGE_CLASSES` uses same pattern
- `src/app/(app)/analytics/page.tsx:108-120` — `STATUS_CONFIG` uses `bg-violet-500`, `bg-blue-500`, etc.

**Problem:** All three pages define their own status color mappings using raw Tailwind palette colors instead of semantic CSS variables (`bg-info`, `bg-warning`, `bg-success`, `bg-destructive`). This makes all 7 themes render the wrong badge colors on those pages. `job-card.tsx` already uses the correct pattern.

**Fix:** Extract a shared `JobStatusBadge` component in `src/components/jobs/job-status-badge.tsx` using semantic colors, then update all three files to use it.

### 2. [MAJOR] Sidebar tooltip clipped by `overflow-y-auto` nav container
**File:** `src/components/layout/sidebar.tsx:184,227`

**Problem:** The `<nav>` element has `overflow-y-auto` which creates a new stacking/clipping context, so the `absolute left-full` tooltip for collapsed nav items is clipped to the nav's bounding box. Users hovering over nav items in collapsed mode cannot see tooltips near the bottom of the nav.

**Fix:** Move `overflow-y-auto` off the `<nav>` wrapper to a wrapping `<div>`, and add `overflow-visible` to the `<nav>` itself so tooltips can escape. The outer scrollable div keeps scrolling behavior; the nav stays visible.

### 3. [MAJOR] Sidebar z-index: mobile menu button and overlay both use `z-40`
**File:** `src/components/layout/sidebar.tsx:130,139`

**Problem:** The "Open menu" button is `z-40` and the backdrop overlay is also `z-40`. When both appear simultaneously, the overlay may render on top of the button, making the button unresponsive. The button should be lower (`z-30`) so the overlay covers it when open.

**Fix:** Change mobile menu button from `z-40` to `z-30`.

### 4. [MAJOR] Jobs page container uses fixed `px-6` — inconsistent with other pages on mobile
**File:** `src/app/(app)/jobs/page.tsx:237`

**Problem:** Container uses `px-6` (24px) while all other app pages use `px-4 sm:px-6`. On narrow mobile viewports the jobs page has wider gutters than dashboard/profile, causing visual inconsistency.

**Fix:** Change `px-6` to `px-4 sm:px-6`.

### 5. [MINOR] Profile contact info truncation missing `min-w-0` parent
**File:** `src/app/(app)/profile/page.tsx:282-295`

**Problem:** `<span className="truncate">` elements sit inside `flex items-center gap-2` containers without `min-w-0` on the container. In flexbox, flex items won't shrink below their `min-content` size by default, so `truncate` may have no effect when the container is tight. This is inconsistent with how dashboard handles the same pattern.

**Fix:** Add `min-w-0` to the three `<div className="flex items-center gap-2 ...">` containers holding truncated contact spans.
