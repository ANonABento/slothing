# [DEAD] Delete /api/jobs and /api/jobs/* route stubs — replaced by /api/opportunities

## Problem

The `/api/jobs/*` route tree was renamed to `/api/opportunities/*` and `next.config.mjs` already 301-redirects the user-facing URLs:

```js
{ source: '/:locale/jobs', destination: '/:locale/opportunities', permanent: true }
{ source: '/:locale/jobs/research/:id', destination: '/:locale/opportunities/:id/research', permanent: true }
{ source: '/:locale/jobs/:id', destination: '/:locale/opportunities/:id', permanent: true }
```

But every old `route.ts` under `apps/web/src/app/api/jobs/` is still on disk, each one a 2–5-line stub that just returns `deprecatedJobsApiResponse(...)`. Worse, two other API routes **re-export** these stubs:

```bash
$ cat apps/web/src/app/api/export/opportunities/route.ts
export { dynamic, GET } from "../jobs/route";
$ cat apps/web/src/app/api/import/opportunities/route.ts
export { dynamic, POST } from "../jobs/route";
```

So `/api/export/opportunities` (a name suggesting the new path) actually serves the **deprecation 410 response** from the old jobs route. That is almost certainly a bug — these are listed in `find apps/web/src/app/api -name route.ts` as `opportunities` endpoints but behave as 410-Gone shells.

## Decision required (do not assume)

The 410 stubs may exist for one of two reasons:
1. **Defensive shim for external API consumers** still hitting old paths → keep, document.
2. **Pure leftover** with no callers → delete.

`grep -rn '/api/jobs' apps/web/src` should reveal who calls them. If only tests + the stubs themselves, option 2.

## Fix

1. Run `grep -rn 'fetch.*\\/api\\/jobs\\b' apps/web/src apps/web/scripts apps/extension 2>/dev/null` and `grep -rn '/api/jobs' apps/web/src/app | grep -v deprecated` to confirm zero **real** callers.
2. If zero callers → delete the entire `apps/web/src/app/api/jobs/` tree and the `deprecated.ts` helper.
3. **Fix the export/import opportunities routes** regardless — they currently 410 their own callers. Either:
   - Implement them properly (mirror the opportunities GET/POST handlers), or
   - Delete them if unused.
4. Update any tests that hit `/api/jobs/...`.
