# [i18n] Root <html lang="en"> hardcoded

## Problem

The root App Router layout hardcoded `<html lang="en">`, so localized routes served the wrong document language.

## Fix

Move document ownership into the locale layout and render `lang`/`dir` from the active locale.

## Files in scope

- `apps/web/src/app/layout.tsx`
- `apps/web/src/app/[locale]/layout.tsx`
- `apps/web/src/i18n/dir.ts`
- `apps/web/src/i18n.ts`
