# [EXT] Extension page: replace text-only feature "visuals" with real screenshots

## Problem

`/en/extension` is a dedicated landing page for the Columbus browser extension, but its three feature blocks have **no real visual** — each "visual" panel is just a centered text label:

```
"LinkedIn, Indeed, Greenhouse, Lever, Workable"
"Recruiter outreach -> review queue"
"Capture -> pending -> apply"
```

For an extension landing page — where visitors specifically want to see what the install will look like — this is the single biggest conversion miss. A browser-extension store visitor (the exact persona who reaches this page after seeing it referenced from Chrome Web Store / Edge / Firefox) expects screenshots of the popup UI, the saved-job state, the recruiter inbox capture, etc.

The hero already includes a hand-rolled SVG mockup of the extension popover (`HeroMockup`) which is good — but the three feature blocks below it fall flat because they're empty placeholders.

## Repro

1. Open `http://localhost:3000/en/extension`
2. Scroll to "Less copying, more deciding" section
3. Right column of each feature card is a grey panel with only a comma-separated text label — no image, no UI mock, no screenshot

## Suspected cause / Root cause

`apps/web/src/app/[locale]/(marketing)/extension/page.tsx:22-43, 137-141` — `featureBlocks[].visual` is a string; the JSX renders it inside a `<div>` instead of an `<Image>` or richer mock.

## Fix

Replace each `visual: string` with one of:

**Option A (recommended for v1 if real screenshots exist):** Real PNG screenshots of the extension UI states. Use `next/image` with explicit width/height to prevent CLS. Add proper alt text:
- "Slothing Columbus popover saving a LinkedIn job posting"
- "Gmail recruiter import view showing pending opportunities"
- "Review queue with three captured roles"

**Option B (interim if no screenshots yet):** Hand-rolled mini-mockups in the same style as the existing `HeroMockup` SVG. Three small variants:
- LinkedIn job card with "Save to Slothing" button overlay
- Gmail-style row converted to a pending opportunity card
- Three-step pipeline visualization (Capture → Pending → Apply) with icons

**Option C (do not ship):** Keep text-only placeholders.

Whichever option: ensure each visual has descriptive `alt` text. The hero's existing `HeroMockup` already has `aria-label` — match that pattern.

Also: extension page hero has a `<Badge>` saying "Columbus browser extension" but nowhere on the page is the product name "Columbus" explained as Slothing's extension. Either drop the codename or explain it once (e.g., "Columbus, the Slothing browser extension, lets you capture jobs anywhere on the web.").

## Acceptance criteria

- [ ] Each of the three feature blocks on `/en/extension` shows a real visual (screenshot OR hand-rolled mock) — not a text-only placeholder

- [ ] If still no screenshots are available, this ticket pivots to Option B (mini-mockups in the HeroMockup style)
