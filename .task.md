# [i18n] 120 hardcoded English JSX strings in headings/dialog titles/labels

## Problem

Visible JSX text — heading text, dialog titles, button labels, table column names — is hardcoded English in 120 places across the app. Users on `/ja`, `/ko`, `/hi`, etc. see English UI chrome mixed with translated body text.

## Repro

```sh
cd apps/web
grep -rEn '>[A-Z][a-z]+ [A-Z][a-z]+' src/components src/app --include="*.tsx" 2>/dev/null | grep -v 't(' | wc -l
# 120
```

(Heuristic catches two-capitalized-word phrases like "Recent Activity", "Upload Failed". Real count is higher when single-word labels are included.)

Plus the marketing navbar `navLinks` array (`apps/web/src/app/[locale]/(marketing)/components/navbar.tsx:11-16`):
```tsx
const navLinks = [
  { name: "Features", href: "#features" },
  { name: "Extension", href: "/extension" },
  { name: "How It Works", href: "#how-it-works" },
  { name: "Pricing", href: "/pricing" },
];
```
None of these route through `t()`.

## Root cause

Components predate the next-intl migration. The translate-messages script can only translate what's in `messages/en.json` — these strings aren't there.

## Fix

Same batching strategy as the a11y migration. Recommended order by user impact:

- **Batch 1 (marketing nav):** `apps/web/src/app/[locale]/(marketing)/components/navbar.tsx` — `navLinks` array → `t("marketing.nav.features")` etc. Add `marketing.nav.{features,extension,howItWorks,pricing}` to `en.json`. (4 strings, but seen by every visitor.)
- **Batch 2 (onboarding):** `ReviewStep`, `UploadStep`, `BuilderStep` → `onboarding.steps.review.title` etc.
- **Batch 3 (interview):** `interview-summary.tsx`, `interview-job-selection.tsx`, `prep-guide-card.tsx` → `interview.summary.*`
- **Batch 4 (jobs/import):** `import-job-*.tsx`, `jobs-toolbar.tsx` (SelectItem labels) → `jobs.import.*`, `jobs.toolbar.filters.*`
- **Batch 5 (dialogs):** all `<DialogTitle>` literals → `dialogs.<feature>.title`
- **Batch 6 (google):** `CalendarSyncButton`, `ContactPicker`, `SaveToDriveButton` → `integrations.google.*`
- **Batch 7 (settings/help):** `help-cards.tsx` ("Using Ollama (Free)"), `data-management.tsx` ("Confirm Import") → `settings.help.*`

After each batch: `pnpm translate:messages` and review the non-en files.

## Acceptance criteria

- [ ] Marketing navbar links translate when locale switches (manual smoke test on `/ja`, `/ko`)
- [ ] All `<DialogTitle>` strings reachable via `t()` lookup
- [ ] `grep -rEn '<h[1-6][^>]*>[A-Z][a-z]+' apps/web/src/components --include="*.tsx" | grep -v '{t(' | wc -l` significantly drops
- [ ] No regression in existing translated keys
- [ ] `pnpm test` and `pnpm type-check` pass

Add corresponding keys to `apps/web/src/messages/en.json`.
