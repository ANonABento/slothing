# [TEST] Add tests for cron auth + email-unsubscribe + auth.ts (high-risk public endpoints, no tests)

## Problem

The repo has solid test density overall — `find apps/web/src -name '*.test.ts' | wc -l` is **180** alongside ~290 non-test files. But several **security-critical, internet-facing, anonymously-reachable** endpoints have **no `.test.ts` neighbor**:

```bash
$ ls apps/web/src/app/api/email/unsubscribe/
route.ts          # no route.test.ts
$ ls apps/web/src/app/api/extension/auth/verify/
route.ts          # no route.test.ts
$ ls apps/web/src/lib/cron-auth*
apps/web/src/lib/cron-auth.ts   # no cron-auth.test.ts
```

These are exactly the files that *must* have tests:
1. `apps/web/src/app/api/email/unsubscribe/route.ts` — token-only auth, sets digest/welcome state for any user whose token leaks. No test verifies that bad tokens are rejected or that the right user is unsubscribed.
2. `apps/web/src/app/api/extension/auth/verify/route.ts` — raw SQL with `db.prepare(... WHERE token = ?).get(token)`. No test for expired-token cleanup or invalid token path.
3. `apps/web/src/lib/cron-auth.ts` — the central guard for **every** `/api/cron/*` route. Cron route tests (`apps/web/src/app/api/cron/email/retry/route.test.ts` etc.) mock `requireCronAuth`, so a regression in `cron-auth.ts` itself (e.g., constant-time-compare removed, env var renamed) would pass all current tests.

## Why this matters

A single bug in `cron-auth.ts` or `unsubscribe-token.ts` is an authentication bypass on a production endpoint. Currently nothing fails CI in that scenario.

## Fix

Add at least:

1. `apps/web/src/lib/cron-auth.test.ts`:
   - Returns 401 when `CRON_SECRET` env unset.
   - Returns 401 when `Authorization` header missing or wrong.
   - Returns `null` (passes) when header matches.
   - Uses constant-time compare (assert via timing or directly that `===` isn't used).

2. `apps/web/src/app/api/email/unsubscribe/route.test.ts`:
   - Missing token → 400.
   - Invalid token (signature fail) → 400 page.
   - Valid token for `topic=daily-digest` → `setDigestEnabled(false, userId)` called with the right userId.
   - Valid welcome token → `markUnsubscribed(userId)` called.

3. `apps/web/src/app/api/extension/auth/verify/route.test.ts`:
   - No header → 401.
   - Unknown token → 401.
   - Expired token → 401 **and** `extension_sessions` row deleted.
   - Valid token → 200 + `last_used_at` updated.

4. (Bonus) Add a `pnpm test:coverage` line in `package.json` (already exists) wired to CI failing under 60% on `apps/web/src/lib/cron-auth.ts`, `apps/web/src/lib/digest/unsubscribe-token.ts`, `apps/web/src/lib/welcome-series/unsubscribe-token.ts`.
