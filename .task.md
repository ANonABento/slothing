# [ATS] Dead code: analyzer.generateRecommendations is never called

## Problem

`apps/web/src/lib/ats/analyzer.ts` defines `generateRecommendations(issues, score)` at line 881-923 but the function is **never exported and never called** from anywhere in the codebase:

```
$ grep -rn "generateRecommendations" apps/web/src/
apps/web/src/lib/ats/analyzer.ts:881:function generateRecommendations(
```

The `ATSAnalysisResult.recommendations` field is populated by `buildRecommendations` in `scoring/index.ts:835-865` instead — `analyzer.ts`'s `analyzeATS` delegates to `scanResume` from the scoring module and returns `.legacy`, which already includes the recommendations from the scoring module.

So `analyzer.ts:881-923` is dead code: ~40 lines of logic that look like they should be wired in but aren't.

This matters because:
1. It's an attractive nuisance — a future contributor might modify it thinking they're changing real behavior.
2. The two implementations of "recommendations" have **diverged** — `analyzer.generateRecommendations` checks `score.keywords < 70` etc. with different thresholds than `scoring/index.buildRecommendations` (also `< 70` but with different wording and ordering). The dead version implies one truth; the live version is another.

## Fix

1. **Delete** `generateRecommendations` from `apps/web/src/lib/ats/analyzer.ts` (lines 881-923).
2. Verify `pnpm type-check` and `pnpm test` still pass.
3. Confirm via grep that no string-based references exist (e.g. docs).

If a future need arises to have a recommendations function in `analyzer.ts`, it should `export` from there and delegate to the scoring module's `buildRecommendations` — but for now, just remove.
