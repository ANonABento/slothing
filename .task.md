# [DX] Stop logging filenames/sizes/hashes per-request in /api/upload — log spam + PII

## Problem

`apps/web/src/app/api/upload/route.ts` calls `console.log` **9 times per upload** with user filenames, byte sizes, parse output, and document IDs:

```bash
$ grep -cE 'console\.log' apps/web/src/app/api/upload/route.ts
9
```

Sample lines:
- `:105  console.log(\`[upload] File received: ${file.name} (${file.size} bytes, ${file.type})\`);` — user filename in plaintext logs
- `:137  console.log(\`[upload] Magic bytes validated: ${magicBytesValid}\`);` — fires on success path
- `:180  console.log(\`[upload] Text extracted: ${extractedText.length} chars\`);`
- `:235/241  console.log(\`...\` ... );` — inside hot loops over parsed entries
- `:299  console.log(\`[upload] Document saved: ${id}\`);`
- `:331/333  console.log(\`[upload] Ingested ${entriesCreated} entries into bank\`);`
- `:350  console.log(\`[upload] Auto-promoted parsed resume into profile\`);`

Same pattern in `apps/web/src/app/api/parse/route.ts:45,72,98,110` (4 more).

## Why this matters

1. **PII surface area**: `file.name` may contain a real human's name (`Jane_Doe_Resume_2026.pdf`). It lands in stdout → Vercel/host logs → any log shipper, with no redaction.
2. **Production log spam**: 9 lines × every upload, no log level, no structured fields.
3. **Inconsistent with the rest of the codebase**: API routes elsewhere use `console.error` only on the catch path. Upload + parse are the outliers.

## Repro

`curl -F file=@resume.pdf http://localhost:3000/api/upload` → tail `/tmp/slothing-dev.log` → 9 `[upload]` lines.

## Root cause

Debug instrumentation added during initial development was never gated.

## Fix

Two options — pick **B**:

**A.** (Minimum) Replace `console.log` with `console.debug` and skip on `process.env.NODE_ENV === "production"`.

**B.** (Recommended) Introduce a thin `apps/web/src/lib/log.ts` with `log.debug(scope, msg, fields?)` that:
- No-ops in production (or routes through `pino`/`console.debug`).
- Strips/hashes filenames (replace `file.name` with `safeBasenameHash(file.name)`).
- Provides one `log.error(scope, error, fields?)` to keep error logs structured.

Then convert `apps/web/src/app/api/upload/route.ts` and `apps/web/src/app/api/parse/route.ts` to use it. Keep the `console.error` paths.
