# [ATS] handleScrapeJob double-fires on Enter+blur — duplicate /api/scanner/scrape-job POSTs

## Problem

The job-URL input in `apps/web/src/components/ats/scanner-form.tsx` (lines 521-535) binds **three** trigger paths to the same `handleScrapeJob()` call with no debounce or in-flight guard beyond a shallow `if (scraping) return`:

```tsx
<input ...
  onBlur={handleScrapeJob}
  onKeyDown={(event) => {
    if (event.key === "Enter") {
      event.preventDefault();
      void handleScrapeJob();
    }
  }}
/>
<Button onClick={handleScrapeJob} ... />
```

When a user types a URL and hits Enter, the keydown handler fires → state update for `scraping = true` → React schedules an update. Then the input loses focus (because Enter triggers form submission semantics in some browsers, or the user clicks Import) → `onBlur` fires synchronously BEFORE the state batch flushes → the `scraping` guard in `handleScrapeJob` reads stale state (`scraping === false`) → **second** fetch is dispatched. The `Button` `onClick` adds a third path. Two POSTs go to `/api/scanner/scrape-job` (rate-limited but each succeeds), and the second response wins — sometimes overwriting `setScrapedJob(null)` from a transient error.

## Repro

1. Open DevTools Network tab, filter `scrape-job`.
2. Paste a LinkedIn URL into the input and **immediately** press Enter.
3. Observe: two `POST /api/scanner/scrape-job` requests fire within ~5ms of each other.
4. If both succeed, only the second's response is rendered. If the first fails and the second succeeds (or vice versa), the UI flickers between error and success states.

## Suspected cause

The `scraping` state is set with `setScraping(true)` inside `handleScrapeJob`, but React batches the state update; concurrent invocations within the same tick see the **previous** `scraping === false`. The `void` in `onKeyDown` further detaches the promise so the second handler doesn't await the first.

## Fix

Replace the `scraping` state guard with a ref-based lock that's read/written synchronously:

```ts
const scrapingRef = useRef(false);

async function handleScrapeJob() {
  const url = jobUrl.trim();
  if (!url || scrapingRef.current) return;
  scrapingRef.current = true;
  setScraping(true);
  try {
    // ... existing body ...
  } finally {
    scrapingRef.current = false;
    setScraping(false);
  }
}
```

Optionally also: drop the `onBlur={handleScrapeJob}` binding entirely (Enter + Button is enough; auto-scrape on blur is surprising UX when the user is tabbing through fields).
