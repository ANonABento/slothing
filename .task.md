# [i18n] French + pt-BR translations have 13 untranslated strings each (Pipeline, Documents, Hackathon, Type, Source...)

## Problem

The `translate:messages` LLM pipeline left common English nouns un-translated in French and Brazilian Portuguese — words like "Pipeline", "Documents", "Type", "Source", "Hackathon", "Status", "Tags", "Manual" appear verbatim. These ARE common loanwords in both languages but in a job-search UI they should be localized: French users expect "Filière" or "Pipeline" depending on context; pt-BR users expect "Tipo" / "Origem" / "Etiquetas" / "Manual"-but-translated-in-context.

## Repro

```sh
cd apps/web
python3 << 'EOF'
import json
def flatten(o, p=''):
    r={}
    for k,v in o.items():
        if isinstance(v,dict): r.update(flatten(v,p+k+'.'))
        else: r[p+k]=v
    return r
en = flatten(json.load(open('src/messages/en.json')))
for loc in ['fr','pt-BR']:
    d = flatten(json.load(open(f'src/messages/{loc}.json')))
    same = [(k,v) for k,v in en.items() if d.get(k)==v and isinstance(v,str)
            and k != 'nav.brand'
            and not k.startswith('settings.language.options.')
            and v.strip() not in {'Slothing','ATS','GitHub','WaterlooWorks','LinkedIn','Indeed','Greenhouse','Lever','Devpost','URL','Kanban','LLM','GPT'}]
    print(f"=== {loc}: {len(same)} suspicious ===")
    for k,v in same: print(f"  {k}: {v!r}")
EOF
```

## Root cause

1. The LLM translator (claude-haiku-4-5 per `docs/i18n-translations.md`) hedged on common English loanwords. Some are legitimate loanwords ("Pipeline" is genuinely used in French biz speak) but the inconsistency is the issue — `nav.groups.pipeline` is "Pipeline" while sibling labels are translated.
2. The `localeOverrides` map in `apps/web/scripts/translate-messages.ts:33` only has `es: { "nav.groups.pipeline": "Pipeline" }` — explicitly pinning it. But there's no equivalent fr/pt-BR overrides for what should be translated.
3. The `passthroughPaths` and `brandTerms` lists shouldn't catch these — they don't. So the LLM just produced English output.

## Fix

Two paths, run both:

### Option 1: Re-run translate:messages with stronger prompt
Update `apps/web/scripts/translate-messages.ts` system prompt to explicitly say: "Translate every value. If a term is commonly used in the target language as a loanword, still provide the localized equivalent — never return the English source verbatim except for brand names in `brandTerms`."

Then:
```sh
cd apps/web
ANTHROPIC_API_KEY=... pnpm translate:messages --locales=fr,pt-BR
```

Add the proven translations directly:
