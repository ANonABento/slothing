# [DX] Stop logging filenames/sizes/hashes per-request in /api/upload — log spam + PII

## Problem

`apps/web/src/app/api/upload/route.ts` calls `console.log` **9 times per upload** with user filenames, byte sizes, parse output, and document IDs.

Same pattern existed in `apps/web/src/app/api/parse/route.ts`.

## Fix

Introduce a thin `apps/web/src/lib/log.ts` with `log.debug(scope, msg, fields?)` that:
- No-ops in production.
- Strips/hashes filenames before logging.
- Provides `log.error(scope, error, fields?)` for structured error logs.

Then convert upload and parse route debug instrumentation to use it while keeping existing `console.error` catch paths.

## Acceptance criteria

- [ ] `grep -cE 'console\.log' apps/web/src/app/api/upload/route.ts` returns **0**.
- [ ] `grep -cE 'console\.log' apps/web/src/app/api/parse/route.ts` returns **0**.
- [ ] No raw `file.name` reaches logs — only a hash or truncated basename.
- [ ] Existing tests pass, plus one new test asserting that a 50-char filename does not appear verbatim in log output.
