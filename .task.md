# [DASH] Profile page has no empty state, no i18n, no auth-error UX — null profile shows blank form

## Problem

`apps/web/src/app/[locale]/(app)/profile/page.tsx` has multiple first-time-user gaps:

1. **No empty state.** When `/api/profile` returns `{ profile: null }` (brand new account, never seeded), the page silently constructs a blank `liveProfile` (line 122-131) and renders an empty form. No "Let's set up your profile" CTA, no "Import from resume" prompt, no link back to onboarding. The completeness card shows 0% with the user staring at empty inputs and zero guidance.

2. **No i18n.** Unlike the dashboard, every single string is hardcoded English: `"Profile"`, `"Overview"`, `"Preferences"`, `"Privacy"`, `"Discard"`, `"Save changes"`, `"No target salary"`, `"Open to recruiter outreach"`, `"Add a headline"`, error/success toasts, etc. Browsing in any other locale still shows English here while the rest of the app translates. (Other pages use `useTranslations` — see `dashboard/page.tsx:81`.)

3. **Auth error UX is wrong.** `loadProfile()` on line 154 throws `new Error("Could not load profile")` on any non-200 response — including 401. So a logged-out user sees the same error message as a 500, with no sign-in link. (Contrast with dashboard's `AUTH_REQUIRED` branch, which is at least correctly typed even if dead today.)

4. **No optimistic save / unsaved-changes guard.** Navigating away with `isDirty=true` silently discards. Users lose work.

5. **Loading state on save is a toast-less spinner.** The button shows a spinner, but no aria-live region announces "Saving…" / "Saved" for screen readers.

## Repro

1. `pnpm dev`
2. `/en/profile` on a fresh database — see blank form, 0% completeness, no onboarding nudge.
3. Switch locale to `/zh/profile` — entire page is still English.
4. With dev tools, block `/api/profile` to force 401 → see "Could not load profile" with no "Sign in" CTA.
5. Edit any field, click a sidebar link → no "discard changes?" prompt.

## Root cause / suspected cause

Profile page was built early when i18n + onboarding flows didn't exist yet, and was never revisited. The dashboard got the i18n treatment; profile is the last hardcoded-English page in `(app)`.

## Fix

1. Add an explicit `profile === null` branch in the render: a `<ProfileEmptyState />` card pointing to "Upload your resume to fill this in" (`/bank`) and "Or fill manually" (which dismisses the empty state and shows the form).
2. Wrap all strings in `useTranslations("profile")` — add `profile` namespace to `messages/en.json` and other locales (use `messages/en.json` keys as the source of truth, copy English strings as defaults).
3. Detect 401 in `loadProfile()`: redirect to `/sign-in?callbackUrl=/${locale}/profile` instead of swallowing into a generic error.
4. Use the existing dirty-form pattern (the dashboard's "Save" + "Discard" pattern is already here — extend to a `useBeforeUnload`/`router.events` guard).
5. Add `aria-live="polite"` to the save status element (line 308-311) so screen readers announce save success.

## Acceptance criteria
