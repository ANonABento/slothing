# [i18n] 241 hardcoded English strings in placeholder/aria-label/title/alt attrs

## Problem

Form accessibility attributes (`placeholder`, `aria-label`, `title`, `alt`) across the app are hardcoded English. Screen reader users on non-English locales hear English labels mixed with translated content — confusing and an a11y violation.

## Repro

```sh
cd apps/web
grep -rEn 'placeholder="[A-Z][a-z]|aria-label="[A-Z][a-z]|title="[A-Z][a-z]|alt="[A-Z][a-z]' src/components src/app --include="*.tsx" 2>/dev/null | grep -v 't(' | wc -l
# 241
```

Concrete examples (`grep -rEn ... | head`):

```
src/components/ats/scanner-form.tsx:495: placeholder="Paste your full resume text here..."
src/components/ats/scanner-form.tsx:570: placeholder="Optional: paste the job description to check keyword matching..."
```

Same pattern, 241 occurrences.

## Root cause

Components were written for the en-only era and never migrated when next-intl shipped. The translate-messages script can't help — these strings aren't in `messages/en.json` at all.

## Fix

Batch migration, broken by component subtree to keep PRs reviewable. Suggested key naming: `<feature>.a11y.<purpose>` (e.g. `bank.search.placeholder`, `bank.search.ariaLabel`).

1. Add an `a11y` sub-tree to each feature in `messages/en.json`:
   ```json
   "bank": {
     "search": {
       "placeholder": "Search your career profile...",
       "ariaLabel": "Search your career profile",
       "clearAriaLabel": "Clear search",
       "shortcutHint": "Press / to focus"
     }
   }
   ```
2. In each component, replace literals with `t("a11y.placeholder")` etc. via `useTranslations("bank.search")`.
3. Don't touch test files or storybook stubs (filtered already).
4. After each component cluster, re-run `pnpm translate:messages` to fan out to all 7 non-en locales.

Suggested batches (rough size ordering):
- **Batch 1 (highest user-visible):** `bank/search-bar.tsx`, `ats/scanner-form.tsx`, `cover-letter/cover-letter-dialog.tsx` (~20 strings)
- **Batch 2 (studio):** `studio/studio-file-panel.tsx`, `studio/version-diff-view.tsx`, `studio/editor-page-settings.tsx`, `studio/template-preview-thumbnail.tsx` (~30)
- **Batch 3 (settings):** `settings/llm-provider-config.tsx`, `settings/prompt-variants-section.tsx`, `settings/opportunity-review-section.tsx`, `settings/what-ai-powers.tsx` (~25)
- **Batch 4 (jobs/opportunities):** `jobs/jobs-hero.tsx`, `jobs/jobs-empty-state.tsx`, `jobs/import-job-edit-form.tsx`, `opportunities/*` (~40)
- **Batch 5 (bank/chunk):** `bank/chunk-expanded-content.tsx` (move bullet up/down/edit/delete × N rows) (~15)
- **Batch 6 (everything else):** keyboard-shortcuts-dialog, google connect, etc. (~110)
