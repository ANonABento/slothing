# [ATS] ats-scans.ts treats async libsql statements as sync — GET /api/ats/scan returns broken history

## Problem

`apps/web/src/lib/db/ats-scans.ts` calls `stmt.run(...)`, `stmt.all(...)`, and `stmt.get(...)` synchronously, but the underlying `LegacyStatement` interface in `apps/web/src/lib/db/index.ts:5-11` defines all three as **async** (returning `Promise<...>`). Result:

- `saveScanResult()` (line 74): `const result = stmt.run(...args)` — `result` is a Promise, so `result.changes === 0` is **always false**, so the guard `throw new Error("Job not found")` never fires. A scan tied to another user's job will silently store nothing on the DB but the route returns a fake `scanId` like the insert succeeded.
- `getScanHistory()` (line 87-93): `const rows = db.prepare(...).all(...) as RawScanRow[]` — `rows` is a Promise, then `.map(mapRow)` blows up at runtime with `TypeError: rows.map is not a function`. The `GET /api/ats/scan` endpoint catches this and returns `{ error: "Failed to fetch scan history" }` with status 500.
- `getScanById()` (line 100-105): same — `row` is a Promise, `if (!row)` is truthy, and the cast `as RawScanRow` then `mapRow(row)` access reads `row.id` on the Promise (likely undefined), returning a broken record.

The existing unit test (`ats-scans.test.ts`) mocks `db.prepare` with **sync** functions returning plain objects, so the test never exercises the real async surface and the bug stays hidden.

## Repro

1. Authenticated user calls `GET /api/ats/scan?limit=20`
2. Server logs `ATS scan history error: TypeError: rows.map is not a function`
3. Client receives `500 {"error":"Failed to fetch scan history"}`

Confirm from code in `apps/web/src/lib/db/index.ts:42-63` — the `prepare()` factory returns `{ all: async, get: async, run: async }`. All other db helpers in the codebase that use the legacy surface `await` these calls; only `ats-scans.ts` does not.

## Root cause

`ats-scans.ts` was likely written against an older sync surface (e.g. `better-sqlite3`) and never updated when the legacy surface was migrated to libsql/Turso async wrappers.

## Fix

1. Make `saveScanResult`, `getScanHistory`, `getScanById` `async`.
2. `await` every `stmt.run(...)`, `.all(...)`, and `.get(...)` call.
3. Update callers: `apps/web/src/app/api/ats/scan/route.ts` line 46 (`await saveScanResult(...)`) and line 73 (`await getScanHistory(...)`).
4. Rewrite `ats-scans.test.ts` so the mock `prepare()` returns async functions (or use `vi.fn().mockResolvedValue({ changes: 1 })`).
5. Add an integration test that calls `GET /api/ats/scan` and asserts a 200 with `{ history: [] }` for an empty user.
