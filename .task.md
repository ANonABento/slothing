# [i18n] No hreflang alternates — search engines see 8 locales as duplicate content

## Problem

Slothing serves 8 locales (en, es, zh-CN, pt-BR, hi, fr, ja, ko) under URL prefixes (`/en`, `/ja`, etc.) but **emits zero hreflang alternates**. Google treats this as duplicate content and can drop non-English variants from SERPs in their target markets.

## Repro

```sh
curl -sI http://localhost:3000/en | grep -iE 'link|content-language'
curl -sI http://localhost:3000/ja | grep -iE 'link|content-language'
# Both return nothing — no Link rel="alternate" headers
```

In code:
```sh
grep -rn "hreflang\|alternates.*languages" apps/web/src/app apps/web/src/lib 2>/dev/null
# Returns nothing
```

Inspecting `apps/web/src/lib/seo.ts` — `buildMetadata()` sets `alternates: { canonical: seo.path }` but **no `languages` map**. `getSiteMetadata()` doesn't set `alternates` at all.

## Root cause

next-intl provides a `generateAlternates` helper, and Next.js Metadata API supports:
```ts
alternates: {
  canonical: "/path",
  languages: { en: "/en/path", ja: "/ja/path", "x-default": "/en/path", ... }
}
```
Neither is wired in. The sitemap (`apps/web/src/app/sitemap.ts`) DOES iterate all locales × routes, so the data is available — just not surfaced as `<link rel="alternate" hreflang>` tags.

## Fix

1. Add `getAlternateLanguages(path: string): Record<string, string>` helper to `apps/web/src/lib/seo.ts`:
   ```ts
   export function getAlternateLanguages(path: string) {
     const langs: Record<string, string> = { "x-default": `/en${path === "/" ? "" : path}` };
     for (const loc of locales) {
       langs[loc] = `/${loc}${path === "/" ? "" : path}`;
     }
     return langs;
   }
   ```
2. Update `buildMetadata()` (line ~165 in `seo.ts`) to include `languages: getAlternateLanguages(seo.path)` in the `alternates` block.
3. Update `getSiteMetadata()` to include `alternates` with `languages` for the home route (currently used by `apps/web/src/app/layout.tsx`).
4. For dynamic per-locale metadata in `[locale]/layout.tsx`, add a `generateMetadata` that wires the current locale's canonical + the language map.
5. Verify rendered output includes:
   ```html
   <link rel="alternate" hreflang="en" href="https://slothing.work/en">
   <link rel="alternate" hreflang="ja" href="https://slothing.work/ja">
   ... (8 locales + x-default)
   ```
